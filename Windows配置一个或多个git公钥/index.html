Windows配置一个或多个git公钥

一、Windows安装git工具

下载地址： https://git-scm.com/download/win

说明：安装为傻瓜式，自行安装

二、 生成SSH密钥

如果你还没有SSH密钥对，你需要先生成。打开Git Bash或命令提示符进入用户目录下的ssh，运行以下命令来生成一个新的SSH密钥(如果你已经有一个，可以跳过这一步)，如果用户目录没有ssh，可以使用命令(mkdir .ssh)创建

命令参数说明： 

-t 即指定密钥的类型，密钥的类型有两种，一种是RSA，一种是DSA 

-b 指定密钥长度。对于RSA密钥，最小要求768位，默认是2048位。命令中的4096指的是RSA密钥长度为4096位，可以忽略  -b 4096使用默认

三、 为不同的账户生成特定的SSH密钥

为了区分不同账户的SSH密钥，你需要为每个账户生成一个不同的SSH密钥。例如，如果你想为GitHub和GitLab生成两个不同的SSH密钥，你可以这样操作

命令参数说明： 

-f 指定将生成的ssh-key保存在该路径文件下

四、 配置Git使用正确的SSH命令

为了让Git知道在连接到特定主机时使用哪个SSH密钥，你需要编辑~/.ssh/config文件(如果不存在，创建它，命令：touch config)。在这个文件中，你可以指定每个主机应该使用的密钥

配置文件参数说明：

    Host : Host可以看作是一个你要识别的模式，对识别的模式，进行配置对应的的主机名和ssh文件

    HostName : 要登录主机的主机名( 服务器IP)

    Port 22 : 端口

    User : 用户名

    IdentityFile : 指明上面User对应的identityFile路径，认证文件

五、 添加SSH公钥到你的账户

登录到你的GitHub、GitLab或其他Git仓库托管服务的账户，并将相应的SSH公钥添加到你的账户设置中。举例(github)：登录github 在右上角头像 —> Setttings —> SSH and GPG keys —>New SSH keys
